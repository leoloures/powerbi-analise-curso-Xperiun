02 · Medidas

Catálogo de medidas

60 medidas em 6 grupos. Nomenclatura: Prefixo | Nome onde C = valor, % = percentual, T = texto, M = switch/dinâmica. Clique em qualquer medida para expandir o DAX e a explicação.

Base Operacional
Tabela: Medidas · métricas-raiz do modelo
C | FaturamentoSUM(ft_vendas[faturamento])+

Tabela: Medidas · Format: R$ #,0

O que faz: soma a receita bruta de todos os itens vendidos no contexto de filtro. Medida-mãe do modelo — praticamente todas as outras métricas derivam direta ou indiretamente dela.

SUM(ft_vendas[faturamento])
Como funciona

Agregação simples. O contexto de filtro (período, produto, categoria) é controlado pelos visuais e slicers — a medida em si não impõe nenhum filtro adicional.

É usada por

Comparativo Ano Anterior (LY)
Tabela: Medidas · padrão SAMEPERIODLASTYEAR para todas as métricas base
C | LY FaturamentoCALCULATE([C | Faturamento], SAMEPERIODLASTYEAR(...))+

Tabela: Medidas · Format: 0

O que faz: faturamento do mesmo período do ano anterior. Âncora dos cálculos YoY — todas as variações percentuais dependem desta (e das suas irmãs LY).

CALCULATE([C | Faturamento], SAMEPERIODLASTYEAR(dim_calendario[Date]))
Como funciona

SAMEPERIODLASTYEAR desloca o contexto de filtro de data um ano para trás. Como Time Intelligence está desativada no modelo, essa função é chamada explicitamente em cada medida LY — ao invés de ser usada via autoDate.

LY Custos · LY Margem Bruta R$ · LY Margem % · LY Ticket Médio · LY Qtd Pedidos · LY Qtd VendidaCALCULATE([métrica], SAMEPERIODLASTYEAR(...))+

Padrão idêntico ao LY Faturamento — mesma lógica aplicada a cada métrica base. São 7 medidas LY no total, uma por métrica relevante.

CALCULATE([[métrica base]], SAMEPERIODLASTYEAR(dim_calendario[Date]))
Variações Year-over-Year
Tabela: Medidas · padrão COALESCE(DIVIDE(...), "-") para evitar erros em anos iniciais
% | YoY FaturamentoCOALESCE(DIVIDE([C | Fat] - [C | LY Fat], [C | LY Fat]), "-")+

Tabela: Medidas · Format: 0.00%

O que faz: variação percentual de faturamento vs. mesmo período do ano anterior. COALESCE retorna "-" quando não há período anterior — evita erros em anos iniciais ou sem dados.

COALESCE(
  DIVIDE([C | Faturamento] - [C | LY Faturamento], [C | LY Faturamento]),
  "-"
)
YoY Custos · YoY Margem R$ · YoY Margem % · YoY Ticket Médio · YoY Pedidos · YoY Qtd · YoY Média PedidosCOALESCE(DIVIDE([métrica] - [LY métrica], [LY métrica]), "-")+

Mesmo padrão YoY para todas as métricas. Algumas usam VAR para calcular o LY inline quando a métrica não tem tabela LY própria (ex: Média Pedidos por dia).

-- Variante com VAR inline (Média Pedidos por dia)
VAR vLY = CALCULATE([C | Média Pedidos por dia], SAMEPERIODLASTYEAR(dim_calendario[Date]))
RETURN COALESCE(DIVIDE([C | Média Pedidos por dia] - vLY, vLY), "-")
Precificação & Margem de Segurança
Tabela: Medidas · análise de preço vs. custo com threshold dinâmico via parâmetro
C | Preço Mín RecomendadoDIVIDE(-[C | Custo un], [Valor % Margem Segurança] - 1)+

Tabela: Medidas

O que faz: preço mínimo para cobrir o custo unitário respeitando a margem de segurança selecionada no slicer. Fórmula: Custo ÷ (1 − margem%).

DIVIDE(
  -[C | Custo un],
  [Valor % Margem Segurança] - 1
)
Como funciona

O parâmetro % Margem Segurança retorna um valor entre 0 e 1. A fórmula aplica a regra de precificação cost-plus invertida: se o custo é R$ 8 e a margem desejada é 20%, o preço mínimo é R$ 8 ÷ (1 − 0,20) = R$ 10.

Depende de

C | Custo un · Valor % Margem Segurança (parâmetro)

% | Var PreçoDIVIDE([Preço un médio] - [Preço Mín], [Preço Mín])+

O que faz: distância entre o preço realizado e o preço mínimo recomendado. Positivo = acima da margem segura; negativo = alerta (produto vendido abaixo do mínimo).

DIVIDE(
  [C | Preço un médio] - [C | Preço Mín Recomendado],
  [C | Preço Mín Recomendado]
)
C | Qtd Produtos Margem BaixaCOALESCE(CALCULATE(COUNTROWS(...), FILTER(..., [% | Var Preço] < 0)), 0)+

O que faz: conta combinações produto×quantidade com preço abaixo do mínimo recomendado. COALESCE garante retorno de 0 (não BLANK) quando todos os produtos estão acima do mínimo.

COALESCE(
  CALCULATE(
    COUNTROWS(VALUES(ft_vendas[id_produto_qtd])),
    FILTER(VALUES(ft_vendas[id_produto_qtd]),
      [% | Var Preço] < 0)
  ), 0
)
T | Qtd Produtos Margem BaixaSWITCH(count, 0 → "Não existem...", 1 → "Existe 1...", "Existem N...")+

O que faz: mensagem contextual para card de aviso — texto muda conforme a quantidade de produtos em alerta. Inclui o percentual de margem selecionado formatado dinamicamente.

SWITCH([C | Qtd Produtos Margem Baixa],
  0, "Não existem itens abaixo da margem de segurança de "
      & FORMAT([Valor % Margem Segurança], "0%") & ".",
  1, "Existe 1 item abaixo da margem...",
  "Existem " & [C | Qtd Produtos Margem Baixa] & " itens abaixo..."
)
M | Switch Tipo MargemSWITCH(SELECTEDVALUE(...), 0 → Margem %, 1 → Margem R$)+

O que faz: retorna margem em % ou R$ conforme a seleção do parâmetro _Campos Tipo Margem. Usado diretamente no YoY de tipo de margem.

SWITCH(
  SELECTEDVALUE('_Campos Tipo Margem'[_Campos Tipo Margem Pedido]),
  0, [% | Margem Bruta %],
  1, [C | Margem Bruta R$]
)
Análise de Pareto
Tabela: Medidas Pareto · concentração de receita por produto com threshold dinâmico
C | Faturamento AcumVAR vFatContexto = [C | Fat] → acumula produtos com Fat ≥ contexto+

Tabela: Medidas Pareto · Format: R$ #,0

O que faz: acumula o faturamento de todos os produtos com faturamento igual ou maior ao do produto no contexto atual — ordena implicitamente do maior para o menor. Padrão clássico de Pareto em DAX.

VAR vFatContexto = [C | Faturamento]
VAR vFatAcumulado =
  CALCULATE([C | Faturamento],
    FILTER(ALLSELECTED(dim_produto),
      [C | Faturamento] >= vFatContexto))
RETURN vFatAcumulado
Como funciona

Para cada produto na tabela, calcula quantos produtos têm faturamento maior ou igual — e soma o faturamento deles. O resultado é um valor crescente: o top produto tem valor = seu faturamento, o segundo acumula os dois, e assim por diante.

% | ParetoDIVIDE([C | Faturamento Acum], CALCULATE([C | Fat], ALLSELECTED(dim_produto)))+

O que faz: percentual acumulado de faturamento — vai de próximo de 0% (produto no topo) até 100% (todos os produtos). A curva 80/20 é lida onde % Pareto ≤ 80%.

VAR vFatTotal = CALCULATE([C | Faturamento], ALLSELECTED(dim_produto))
RETURN DIVIDE([C | Faturamento Acum], vFatTotal)
C | Faturamento Par % · C | Qtd Produtos Pareto · % | Qtd Produtos ParetoFILTER(ALLSELECTED(dim_produto), [% | Pareto] ≤ threshold)+

O que fazem: métricas para o KPI "N% dos produtos geram X% do faturamento". Filtram a lista de produtos ao corte Pareto selecionado (ex: top 80%) e calculam o faturamento acumulado e a proporção do catálogo.

-- C | Faturamento Par %
CALCULATE([C | Faturamento],
  FILTER(ALLSELECTED(dim_produto), [% | Pareto] <= [Valor Seleção Pareto]))

-- C | Qtd Produtos Pareto
CALCULATE(COUNTROWS(
  FILTER(ALLSELECTED(dim_produto), [% | Pareto] <= [Valor Seleção Pareto])))
C | Ranking ParetoIF(ISINSCOPE(dim_produto[produto e sabor]), RANKX(ALL(dim_produto), [C | Fat],, DESC))+

O que faz: rank de faturamento por produto. ISINSCOPE garante que o rank só aparece no nível de produto, não em linhas de total ou subtotal.

IF(ISINSCOPE(dim_produto[produto e sabor]),
  RANKX(ALL(dim_produto), [C | Faturamento],, DESC)
)
Filtro ParetoIF([% | Pareto] ≤ [Valor Seleção Pareto], 1, 0)+

O que faz: flag 1/0 para filtrar a tabela de produtos ao corte Pareto. Usada como filtro visual em tabelas e slicers.

IF([% | Pareto] <= [Valor Seleção Pareto], 1, 0)
Dynamic Fields
Tabela: Medidas DF · switches para visuais com campos selecionáveis pelo usuário
M | Switch Evol Col / LinSWITCH(SELECTEDVALUE(_Campos_Evol_Col[Pedido]), 0 → Fat, 1 → Custos ... 7 → LY Margem%)+

Tabela: Medidas DF

O que fazem: direcionam para uma das 8 métricas conforme o seletor de campo de evolução (Faturamento, Custos, Margem R$, Margem %, e as 4 versões LY correspondentes). Um switch por eixo (coluna e linha).

SWITCH(SELECTEDVALUE(_Campos_Evol_Col[_Campos_Evol_Col Pedido]),
  0, [C | Faturamento],    1, [C | Custos],
  2, [C | Margem Bruta R$], 3, [% | Margem Bruta %],

O que fazem: YoY dinâmico — calcula a variação percentual da métrica selecionada vs. ano anterior. O LY é calculado inline via VAR para evitar dependência de uma tabela LY específica.

VAR vLY = CALCULATE([M | Switch Evol Col],

RETURN
  COALESCE(DIVIDE([M | Switch Evol Col] - vLY, vLY), "-")
M | Switch Medidas ProdutosSWITCH(0–5 → Fat, Custos, Margem R$, Margem %, Qtd Vendida, Qtd Pedidos)+

O que faz: métrica dinâmica para o visual de análise por produto — 6 opções selecionáveis via parâmetro _Campos Medidas Produtos.

SWITCH(SELECTEDVALUE(_Campos_Medidas_Produtos[Pedido]),
  0, [C | Faturamento],     1, [C | Custos],
  2, [C | Margem Bruta R$], 3, [% | Margem Bruta %],
  4, [C | Qtd Vendida Produtos], 5, [C | Qtd Pedidos]
)
M | Switch Dispersão X / YSWITCH(0–6 → 7 métricas para eixos do scatter plot)+

O que fazem: eixos X e Y do scatter plot — 7 métricas disponíveis (Faturamento, Custos, Margem %, Margem R$, Qtd Pedidos, Qtd Vendida, Ticket Médio). Permite exploração livre de correlações entre quaisquer dois indicadores.

T | Switch nome parâmetro ProdutoSWITCH(0–3 → SELECTEDVALUE(dim_produto[campo correspondente]))+

O que faz: retorna o nome do produto no nível de granularidade selecionado — produto simplif, produto+sabor, só sabor, ou quantidade.

SWITCH(SELECTEDVALUE(_Campos_Produto[Pedido]),
  0, SELECTEDVALUE(dim_produto[produto simplif]),
  1, SELECTEDVALUE(dim_produto[produto e sabor]),
  2, SELECTEDVALUE(dim_produto[sabor]),

04 · Dependências

Grafo de dependências

Quem depende de quem entre as medidas. Alterar C | Faturamento afeta ~12 medidas a jusante — cuidado em refatores.

Árvore de C | Preço Mín Recomendado (medida mais profunda):

C | Preço Mín Recomendado ├── C | Custo un │ ├── C | Custos │ │ └── ft_vendas[custo] │ └── C | Qtd Vendida Produtos │ └── ft_vendas[quantidade] └── Valor % Margem Segurança └── '% Margem Segurança'[% Margem Segurança] (parâmetro) % | Var Preço ├── C | Preço un médio │ ├── C | Faturamento → ft_vendas[faturamento] │ └── C | Qtd Vendida Produtos → ft_vendas[quantidade] └── C | Preço Mín Recomendado (ver acima) % | Pareto ├── C | Faturamento Acum │ └── C | Faturamento → ft_vendas[faturamento] └── C | Faturamento (via ALLSELECTED) % | YoY Evol Col └── M | Switch Evol Col └── → qualquer das 8 métricas base conforme seleção do parâmetro

Reverse — impacto das medidas-base:

C | FaturamentoMais referenciada

Usada por: C | LY Faturamento · % | YoY Faturamento · % | Margem Bruta % · C | Ticket Médio · C | Preço un médio · C | Faturamento Acum · % | Pareto · C | Faturamento Par % · M | Switch Evol Col · M | Switch Evol Lin · M | Switch Dispersão X · M | Switch Dispersão Y

Alterar a definição de Faturamento afeta 12 medidas a jusante. É a medida mais crítica do modelo.

C | Margem Bruta R$Alta dependência

Usada por: % | Margem Bruta % · C | LY Margem Bruta R$ · % | YoY Margem R$ · M | Switch Tipo Margem · M | Switch Evol Col · M | Switch Evol Lin · M | Switch Medidas Produtos · M | Switch Dispersão X/Y

Afeta 8+ medidas. Depende de C | Faturamento e C | Custos.

Usada por: C | Preço Mín Recomendado

% | ParetoMedida-base do Pareto

Usada por: C | Faturamento Par % · C | Qtd Produtos Pareto · % | Qtd Produtos Pareto · Filtro Pareto
